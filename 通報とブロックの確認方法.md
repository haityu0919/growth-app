# 通報とブロックの確認方法

このプロジェクトでは、開発中に中身を把握しやすいように、通報とブロックを **JSONファイル** に保存します。

## 保存場所

| ファイル | 内容 |
|----------|------|
| `token-server/data/reports.json` | 通報した相手・理由・日時 |
| `token-server/data/blocks.json` | ブロックした相手・日時 |

## 確認手順

1. `token-server/start-server.bat` を起動したままにする
2. アプリで通報またはブロックを試す
3. 次のフォルダを開く

```text
C:\Users\matsuda\Documents\growth-app\token-server\data
```

4. `reports.json` または `blocks.json` をメモ帳で開く

## 保存例

### reports.json

```json
[
  {
    "id": "自動生成ID",
    "userId": "demo-user",
    "targetId": "friend-1",
    "targetName": "ゲストA",
    "reason": "暴言・嫌がらせ",
    "createdAt": "2026-05-18T12:00:00.000Z"
  }
]
```

### blocks.json

```json
[
  {
    "id": "自動生成ID",
    "userId": "demo-user",
    "targetId": "friend-1",
    "targetName": "ゲストA",
    "createdAt": "2026-05-18T12:00:00.000Z"
  }
]
```

## ブロックした相手に起きること

- チャットできません
- 選択式通話の一覧に出ません
- ラジオ配信の一覧に出ません

ランダム通話での再マッチ除外は、本番のユーザー管理を Supabase に移したあと、サーバー側で完全に制御します。
