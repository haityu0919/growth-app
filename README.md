個人開発向けプロジェクト。5担当＋統括の仮想チーム体制で進める。

## アプリ概要（確定）

| 項目 | 内容 |
|------|------|
| **アプリ名** | **だれトーク** |
| **略称** | **ダレト** |
| **参考アプリ** | 斉藤さん（匿名・不特定のユーザーとのエンタメ型SNS） |
| **プラットフォーム** | **iPhone・Android 両方** |
| **コア機能①** | **ランダム通話** … ボタン1つで不特定の相手と通話（基本は音声） |
| **コア機能②** | **音声ラジオ配信** … 聴く／配信する、どちらも可能（ビデオ配信は禁止） |

### キャッチコピー・文案（案）

- キャッチコピー：「誰と話す？ 誰の声を聴く？」
- 起動画面：「今日も、だれかとつながろう」
- ランダム通話：「だれトークする」
- ラジオ配信：「だれラジオを聴く／配信する」（音声のみ）

### 画面構成（案）

```
だれトーク
├── ホーム
│   ├── 【ランダム通話】 … 1タップで誰かと通話
│   └── 【音声ラジオ配信】 … いま配信中一覧 → 聴く / 配信を始める
├── 通話中画面
└── 配信中画面（聴取 / 配信者）
```

### 音声のみのルール（確定）

- **ビデオ通話は完全廃止**する
- **ラジオ配信も音声のみ**にする
- ビデオ配信、画像送信、URL送信は初期版では扱わない
- 審査と一人運営を考え、安全対策を優先する

### ラジオ配信（確定）

- **聴く**（視聴）と **配信する**（放送）の **どちらも** MVP に含める
- ラジオは **音声配信のみ**。ビデオ配信は禁止
- ホームから配信中一覧 → 聴取／新規配信開始、の両導線を用意する

### 収益化・プレミアム（準備中）

**広告表示ルール**

- 1分以上通話した場合、通話終了後に広告を表示する
- ラジオを30秒以上聴いた場合、聴取終了後に広告を表示する
- 広告は表示開始から5秒後にスキップ（閉じる）可能
- プレミアム会員は広告非表示
- 初回リリースでは課金を開始せず、プレミアムは準備中として表示する

**フィルター仕様**

- プレミアム会員は、年齢・性別で相手を絞り込める
- 例: `10代` + `女性`
- ランダム通話と選択式通話の両方で使う
- ランダム通話では、サーバー側で条件に合う待機相手だけとマッチングする
- 無料ユーザーはフィルターを使えない

**月額800円の価値説明**

- 広告なしで会話に集中できる
- 年齢・性別フィルターで話したい相手を探しやすい
- 優先マッチングでつながりやすさを上げる
- お気に入り枠と通話履歴30件保存で、よく使う人ほど便利になる

### 技術方針（確定・AI開発向け）

**方針:** プログラム経験なしでも AI がほぼコードを書ける組み合わせを採用する。

| 役割 | 選定 | 理由（補足） |
|------|------|--------------|
| **アプリ本体** | **Expo（React Native）** | iPhone・Android を1つのコードで。AI の学習データが豊富で Cursor との相性が良い |
| **通話・ラジオ** | **LiveKit** | 1対1通話もラジオ（音声ルーム）も同じ仕組みで扱える。SDK とドキュメントが整っている |
| **バックエンド** | **Supabase** | ユーザー管理・データ保存・マッチング用 DB。無料枠あり、AI が扱いやすい |

**ユーザー（開発者）がすること**

- Cursor に **日本語で指示** する（例:「ホーム画面を作って」）
- AI の提案を **承認** する
- スマホで **動作確認** する（Expo Go アプリを使用）
- エラー画面が出たら **スクショやメッセージを AI に貼る**

**AI（Cursor）がすること**

- コードの作成・修正
- ターミナルコマンドの実行（承認後）
- エラー調査・修正
- 専門用語の補足説明

### AI開発の進め方

1. **小さく依頼** … 「全部作って」より「ホーム画面だけ」「通話ボタンだけ」
2. **承認してから実行** … ファイル作成・コマンドは AI が内容を示してから
3. **動かして確認** … 1機能ごとにスマホでテスト
4. **うまくいかなければ貼る** … エラー文・スクショをチャットに

**Cursor への依頼例**

- 「だれトークのホーム画面を作って。ランダム通話とラジオ配信の2ボタン」
- 「LiveKit で音声通話をつないで。まずは2人でテストできるように」
- 「エラーが出た。ログ: （ここに貼る）」
