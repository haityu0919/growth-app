## 2. アプリ総括（チーム5名）

### ジョブズ（統括）

**だれトークはいまどこまで来ているか**  
初回リリースの「形」はほぼ揃った。画面・法務・安全機能・マッチング・ブランド・ストア用ビジュアルまで進み、**残る核心は1つだけ：開発版での音声通話の実機確認**。Apple Developer は申請済みだが **承認未着**。その間にサーバークラウド化とストア文案を進められる。

**今日の位置づけ**  
「作る」より「出す準備」と「軽くする」が進んだ一日。ストア画像の確定は大きい。次のゴールは承認メール → iOS ビルド → 2台テスト。

### はるか（リサーチ）

**ユーザーから見たアプリ像**  
「夜に、怖くない匿名の声の雑談」。ラベンダードーンとくもりんで、従来の匿名アプリより不安が下がる見た目になった。ストア5枚は、安心・簡単・匿名・待ち時間・安全の順で心理のハードルを下げている。

**リスク**  
マッチングはまだ PC 上の token-server 前提。クラウド化しないと、外出先や TestFlight テストがつらい。10代向け注意・通報は実装済みだが、音声体験が未検証のままストアに出すのは避けたい。

### 風間（冒頭構成）

**体験の流れ**  
(1) スプラッシュ（新：マークのみ）→ (2) LaunchScreen（アプリ名・キャッチ）→ (3) 規約同意 → (4) ホーム「ランダム通話」→ (5) くもりん待機 → (6) 通話。  
Expo Go では (6) の音声まで届かないが、**入口〜待機まで一連で確認できる**状態。

**ストア画像との整合**  
掲載順は「魅力 → 使い方 → 匿名 → くもりん → 安全」で、初回起動後の体験と矛盾しない。FAQ（Expo Go vs 開発版）は承認待ちのあいだに書くとよい。

### 匠（ブラッシュアップ）

**技術・UI**  
`theme.js` 集約、LiveKit 遅延読込、英数字 bat 起動、PNG 最適化スクリプトまで揃った。新アイコンは文字なしでホーム画面でも読みやすい。`apply-brand-assets.bat` 実行後は `optimize-pngs.bat` → Expo 再起動が必要。

**未完了の実装タスク**  
- `assets` への新画像コピー（松田さん PC で bat 実行）  
- サーバー URL をクラウド HTTPS に差し替え（`config.js`）  
- 開発版ビルド後の LiveKit 実機テスト

### こと（台本）

**言葉・ストア**  
キャッチは **「声でつながる、きもち近づく」** に起動画面とストアで揃えた。掲載画像の「顔出しなし・ビデオなし」は1枚に2メッセージで重複していたため、**「匿名のまま、声だけで話せる」** に修正して確定。

**ストア説明の芯**  
匿名ランダム音声／ビデオなし／通報・ブロック／10代は注意表示。プライバシーポリシー URL は審査前に Web 公開が必要。
