## 2. チーム話し合い

### 争点と結論

| テーマ | 結論 |
|--------|------|
| Android 先行 vs iOS 先行 | iPhone 2台のみ → Apple Developer 加入後 iOS 一択 |
| くもりん背景の溶け込み | 画像は変えず、サイズ拡大のみ（松田さん判断） |
| Expo Go の位置づけ | UI・マッチング確認用。音声は開発版のみ（全員一致） |

### 良かった点

- **ジョブズ:** マッチング実機確認、ブランド確定で初回リリースに近づいた
- **はるか:** Expo Go で検証ループが回せる。くもりんで「怖くない待ち時間」
- **風間:** 起動→規約→マッチングの流れが体感できる
- **匠:** ロゴ02とくもりんで世界観統一。theme.js とアセット整理
- **こと:** 「声だけで安心して」がビジュアルでも伝わる

### 改善点・懸念

- **ジョブズ:** 音声未検証が最大リスク
- **はるか:** マッチングは PC 同一 Wi-Fi 依存
- **風間:** くもりん画像の縁が端末によって気になる場合あり
- **匠:** PNG サイズがやや大きい。リリース前に軽量化
- **こと:** Expo Go と開発版の違いは FAQ 化が必要

### 今後の展望

| 期間 | 方針 |
|------|------|
| 短期（2週間） | Apple Developer 加入 → iOS 開発版 → 音声テスト |
| 中期（1〜2か月） | サーバークラウド化、TestFlight、ストア素材 |
| 長期（3か月〜） | 初回リリース → 第2段階機能 |

### チーム結論

1. ロゴ **02 ボイス・バブル**、マスコット **くもりん** を正式採用
2. **Apple Developer Program** に加入し iOS 開発版から音声テスト
3. サーバー常時稼働は初回リリース前に必須
